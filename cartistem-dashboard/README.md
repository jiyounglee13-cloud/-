# CARTISTEM 논문 핵심 메시지 대시보드

동종 제대혈 유래 중간엽 줄기세포(hUCB-MSC)와 4% 히알루론산 하이드로젤 복합체
**CARTISTEM®** 의 무릎 연골 재생 관련 학술 논문을, 독자가 5분 이내에 핵심을 파악할 수
있도록 **표 중심·사실 기반**으로 요약한 정적(static) 대시보드입니다.

> 의학·생명과학 논문 핵심 메시지 요약 프롬프트(표 A~E, 프리미엄 포맷)를 그대로 구현했습니다.
> 모든 수치·통계량은 **원문(초록·결과·고찰)에 기재된 값을 그대로 전사**했으며, 추측·과장·외삽을
> 배제했습니다. 정보가 없는 항목은 `원문 미보고`로 표기합니다.

## 실행 방법

별도 빌드·서버·의존성이 필요 없습니다. 데이터는 `data.js`에 내장되어 있어
`file://` 로도 동작합니다.

```bash
# 1) 파일을 브라우저로 직접 열기
open cartistem-dashboard/index.html        # macOS
xdg-open cartistem-dashboard/index.html    # Linux

# 2) 또는 간단한 로컬 서버
cd cartistem-dashboard && python3 -m http.server 8080
# http://localhost:8080
```

## 구성

| 파일 | 역할 |
| --- | --- |
| `index.html` | 페이지 골격(헤더·개요·종합 메시지·통계·필터·카드 목록) |
| `styles.css` | 프리미엄 클리니컬 테마(반응형 + 인쇄/PDF 최적화) |
| `app.js` | 데이터 렌더링, 검색·필터·정렬, 카드 토글, 인쇄 |
| `data.js` | 논문 데이터(표 A~E). `window.PAPERS`, `window.CARTISTEM_META` |

## 출력 포맷 — 표 5종

각 논문 카드는 프롬프트 명세의 5개 표로 구성됩니다. **위계 규칙**에 따라
표 D(핵심 메시지)를 카드 상단에 항상 노출·강조하고, 표 C의 수치를 D의 근거로
상호 참조합니다.

| 표 | 구성 |
| --- | --- |
| **A. 논문 식별** | 제목 / 저자 / 저널·연도 / 연구 설계 / 근거 수준 / 식별자(PMID·등록번호·IF) |
| **B. 연구 개요** | 연구 질문 / 대상·표본 수 / 중재·비교 / 주요 평가변수 / 추적 기간 |
| **C. 핵심 정량 결과** | 평가변수 / 결과군 / 대조군 / 효과크기 / 유의성(p·CI) |
| **D. 핵심 메시지** | 핵심 함의 + 각 메시지의 근거 데이터(표 C·B·E 참조) |
| **E. 한계·후속 연구** | 한계 / 결과 해석에 미치는 영향 / 후속 연구 제언 |

## 대시보드 기능

- **코퍼스 종합 메시지** — 전체 논문을 가로지르는 핵심 함의를 상단에 우선 배치
- **요약 통계** — 수록 편수, RCT+메타분석, 임상·중개 연구, 근거 수준 분포
- **검색** — 제목·저자·저널·결과 수치·메시지 전문 검색
- **필터** — 연구 유형(RCT·메타분석·코호트·증례·술기·경제성·관련·참고),
  근거 수준(최상위·상위·중간·하위·맥락 의존)
- **정렬** — 논문 번호 / 연도 / 근거 수준
- **인쇄/PDF** — 인쇄 시 전 카드를 펼쳐 보고서 형태로 출력

## 수록 논문(31편)

CARTISTEM 핵심 임상·중개 연구 28편(3상 RCT, 7년 추적 1상, HTO 병용,
BMAC·미세천공 비교, 체계적 문헌고찰·메타분석, 비용효과 분석 등)과
관련 연구 1편, 방법론 참고문헌 2편(MRI 등급 과소평가·연골 병변 측정)을 포함합니다.
참고·관련 문헌은 카드의 태그(`방법론 참고`, `관련 연구`)로 구분됩니다.

## 데이터 추가/수정 방법

`data.js`의 `window.PAPERS` 배열에 동일한 스키마로 객체를 추가하면 카드가 자동 생성됩니다.

```js
{
  id: "p31", num: 31,
  category: "cohort",        // rct|review|cohort|caseseries|casereport|technique|economics|related|reference
  evidence: "mid",           // top|high|mid|low|context
  featured: false,           // 상단 강조 여부
  title: "...", authors: "...", journal: "...", year: 2025,
  design: "...", evidenceLabel: "근거수준 ...",
  pmid: "...", reg: "...", impact: "IF ...",
  tableB: { question, population, intervention, outcomes, followup },
  tableC: [ { v: "평가변수", r: "결과군", c: "대조군", e: "효과크기", s: "p·CI" } ],
  tableD: [ { m: "핵심 메시지", b: "근거(표 C/B/E 참조)" } ],
  tableE: [ { l: "한계", i: "해석 영향", f: "후속 제언" } ]
}
```

> 원문에 없는 데이터·결론·권고는 생성하지 않으며, 정보 부재 시 `원문 미보고`로 표기합니다.
